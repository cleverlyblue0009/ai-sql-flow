# Experiment Setup Guide
## IEEE Access Research Suite — AI-SQL-Flow

**Estimated runtime:** 15–45 minutes (depending on hardware)  
**Estimated disk:** ~500 MB (datasets + results + figures)

---

## Prerequisites

### Software
- Python 3.9 or higher (tested on 3.14)
- pip ≥ 22.0
- Git

### Hardware (minimum)
- 4 GB RAM (8+ GB recommended for 100K-row scalability test)
- 4-core CPU (all experiments are single-threaded; more cores do not accelerate)
- 1 GB free disk space

---

## Quick Start

### Option A: Bash (Linux/macOS/WSL)

```bash
git clone <repository-url>
cd ai-sql-flow
git checkout research-experiments-ieee-access

# Install dependencies
pip install -r requirements.txt
pip install -r research_assets/reproducibility/requirements_experiments.txt

# Run all experiments
bash research_assets/reproducibility/run_all_experiments.sh

# Skip slow experiments (scalability 100K, cross-validation 10x5-fold)
bash research_assets/reproducibility/run_all_experiments.sh --skip-slow
```

### Option B: Windows PowerShell

```powershell
git clone <repository-url>
cd ai-sql-flow
git checkout research-experiments-ieee-access

# Install dependencies
pip install -r requirements.txt
pip install -r research_assets\reproducibility\requirements_experiments.txt

# Run all experiments
$env:PYTHONPATH = "$(Get-Location)"
python research_assets\experiments\run_all_experiments.py

# Skip slow experiments
python research_assets\experiments\run_all_experiments.py --skip-slow
```

### Option C: Run Individual Experiments

```bash
cd ai-sql-flow
export PYTHONPATH=.

# Step 1: Generate datasets (required before all others)
python research_assets/experiments/generate_research_datasets.py

# Step 2: Run any experiment
python research_assets/experiments/run_ablation_study.py
python research_assets/experiments/run_baseline_comparison.py
python research_assets/experiments/run_scalability_test.py   # slow: ~5 min
python research_assets/experiments/run_robustness_test.py
python research_assets/experiments/run_confidence_analysis.py
python research_assets/experiments/run_false_positive_analysis.py
python research_assets/experiments/run_cross_validation.py  # slow: ~10 min
python research_assets/experiments/run_enterprise_case_study.py
python research_assets/experiments/run_security_validation.py

# Step 3: Generate figures (requires all result JSONs)
python research_assets/experiments/generate_figures.py
```

## Troubleshooting

### `ModuleNotFoundError: No module named 'psutil'`
```bash
pip install psutil
```

### `ModuleNotFoundError: No module named 'research_assets'`
Ensure `PYTHONPATH` includes the project root:
```bash
export PYTHONPATH=/path/to/ai-sql-flow
# or run from the project root
cd ai-sql-flow && python research_assets/experiments/...
```

### `ValueError: output array is read-only`
This was fixed in the current version. If encountered, ensure you have the latest commit from `research-experiments-ieee-access` branch.

### `AttributeError: 'numpy.ndarray' object has no attribute 'ptp'`
This was fixed in the current version. `ptp()` deprecated in NumPy 2.0 — replaced with `max() - min()`.

### Scalability test runs out of memory at 100K rows
The 100K cosine similarity matrix requires ~4GB peak RAM. If unavailable:
```bash
python research_assets/experiments/run_all_experiments.py --skip-slow
```
This skips both the scalability test and cross-validation.

### Figures not generated
Ensure all result JSON files exist before running `generate_figures.py`:
```bash
ls research_assets/results/ablation/
ls research_assets/results/baseline/
# ... all 9 result directories
```

## Verifying Results

After execution, verify key metrics match expected values from REPRODUCIBILITY_REPORT.md:

```python
import json

# Example verification
with open("research_assets/results/ablation/ablation_results.json") as f:
    r = json.load(f)

full_f1 = r["aggregate"]["full_system"]["mean_f1"]
assert abs(full_f1 - 0.3119) < 0.001, f"Expected 0.3119, got {full_f1}"
print("Ablation verification PASSED")
```

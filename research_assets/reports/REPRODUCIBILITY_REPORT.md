## 1. Reproducibility Guarantees

All experiments in this suite are designed to produce bit-identical results on re-execution, subject to the conditions in §2.

### 1.1 Random Seed Control

```python
# experiment_config.py
RANDOM_SEED = 42

# All experiments
np.random.seed(RANDOM_SEED)           # global numpy
rng = np.random.default_rng(RANDOM_SEED)  # per-call generator

# All sklearn models
IsolationForest(random_state=42)
LocalOutlierFactor(random_state=42)
KNNImputer()                          # deterministic — no RNG
```

### 1.2 Dataset Reproducibility

Datasets are generated once by `generate_research_datasets.py` and saved as static CSV files. All subsequent experiments read from these fixed files. Ground-truth masks are reconstructed from the saved `_ground_truth.json` files using the same `np.random.default_rng(42)` call.

### 1.3 Output Determinism

All JSON output files are written with `indent=2` and dictionary key ordering preserved by Python 3.7+ dict insertion order.

### 2.1 Version Sensitivity

Float precision differences between NumPy versions may cause sub-0.001 differences in metric values. Metric rounding to 4 decimal places absorbs these differences. Functional results (which rows are flagged) are fully deterministic given the same seed.

## 4. Common Reproducibility Pitfalls

1. **Running experiments out of order:** Experiments 2–10 read from dataset files generated by Experiment 1. Always run `generate_research_datasets.py` first.
2. **Missing packages:** Run `pip install -r requirements_experiments.txt` before executing.
3. **psutil import error:** `pip install psutil` — not included in all base environments.
4. **TF-IDF vocabulary randomness:** TF-IDF `max_features=500` is deterministic given sorted vocabulary — no RNG needed.
5. **Scalability variance at 50K rows:** OS memory pressure causes timing variance (±17%). This is expected and documented.
